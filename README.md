# 🎮 ГОСУДАРСТВО ОНЛАЙН - Telegram Bot 3.0

## Полная архитектурная переписка на асинхронный aiogram 3.x + aiosqlite

Переработанная версия Telegram-бота политической симуляции с полным переходом на **современные асинхронные библиотеки**.

- **🎪 Полная асинхронность** - aiogram 3.x + aiosqlite, без блокирующих операций

## 🏗️ Архитектурное решение

### Database Layer (`database.py`)

Полностью асинхронный слой с aiosqlite:

```python
# Все операции async/await
user = await db.get_user(user_id)
await db.update_user(user_id, balance=5000)
org = await db.get_organization("Правительство")
```

**30+ таблиц**, включая:

- `users` - профили игроков
- `organizations` - гос. структуры
- `elections` - выборы и голосования
- `businesses` - бизнесы и доходы
- `loans` - кредиты
- `government_system` - состояние правления

### FSM & States (`states.py`)

Иерархическая система состояний для каждой фичи:

```python
class MainStates(StatesGroup):
    main_menu = State()
    waiting_choice = State()

class ElectionStates(StatesGroup):
    global_lock = State()      # Режим выборов
    voting_menu = State()
    campaign = State()

class PresidentStates(StatesGroup):
    admin_panel = State()
    appointing_position = State()
```

### Middleware Chain (`middlewares.py`)

Взаимно вложенная система валидации:

```
EnsureUserMiddleware     # Создаёт/обновляет пользователя
    ↓
GlobalLockMiddleware    # Проверяет режим выборов
    ↓
Handler
```

**GlobalLockMiddleware** - ключевой элемент:

- Если нет президента AND есть активные выборы:
    - БЛОКИРУЕТ все кроме: голосования, просмотра партий, помощи
    - Все остальные callback даже не доходят до handler

### CallbackData Factories (`keyboards.py`)

Типизированные callback'и вместо строк:

```python
@dataclass
class OrgCallback(CallbackData, prefix="org"):
    action: str          # "view_org", "apply_org", etc
    org_id: int
    org_name: str

# Использование:
callback_data=OrgCallback(action="view_org", org_id=1, org_name="Правительство").pack()
```

## 🎮 Основные сценарии использования

### Сценарий 1: Первые выборы

1. Бот запускается → БД пуста
2. `/start` срабатывает → нет президента
3. Проверка: `await db.check_has_president()` → False
4. Запускаем выборы: `await db.start_election(org_id, "President", 30)`
5. **GlobalLockMiddleware** активируется:
    - Все кнопки КРОМЕ выборов не работают
    - Игроки видят: "🔒 РЕЖИМ ВЫБОРОВ АКТИВЕН!"
6. Когда президент выбран → режим нормализуется

### Сценарий 2: Произвольное в админ-панели

1. Президент нажимает `/admin` или `president_admin_panel`
2. Проверка: `gov['current_leader_id'] == user_id` → True
3. Показываем панель с опциями:
    - 👤 Назначение должностей
    - ✏️ Переименование должостей
    - ➕ Создание новых должостей
    - 🏛️ Смена формы правления (Демократия / Монархия / Диктатура)
4. Все изменения немедленно обновляют БД

### Сценарий 3: Революция

1. Игрок платит $100,000 для запуска революции
2. Пишет манифест
3. Революция прясыпается, собирает сторонников
4. Каждый может агитировать: +1 сторонник за призыв
5. При 50+ сторонниках → голосование
6. Если голосов больше → смена правительства и лидера

### Сценарий 4: ФБР наблюдение

1. Агент ФБР нажимает "📡 Перехват"
2. Видит все сообщения в игре (заготовка)
3. Может отследить конкретного игрока
4. История его сообщений, контакты, маршрут
5. Может запустить операцию: арест, шантаж, скандал

## 📊 Экономическая система

### Налоги (ежедневно)

Каждый день в 00:00 запускается `run_daily_economy_cycle()`:

```python
# Налог на доход
tax = max(balance * 0.0035, $30)  # 0.35% или минимум $30

# Налог на имущество
property_tax = property_value * 0.00025  # 0.025%

# Налог на бизнес
business_tax = daily_income * 0.001  # 0.1%

# Штраф за просрочку кредита
penalty = loan_amount * 0.01  # 1% в день
```

### Зарплаты

Каждый день государство выплачивает зарплаты:

- Полицейский: $500/день
- Врач: $600/день
- Судья: $800/день
- Чиновник: $450/день

### Кредиты

- Процент: 15% в месяц
- Максимум: $50,000
- Минимум: $1,000
- Штраф за просрочку: -10 репутации + 1% в день

### Бизнес

- Стартовый бюджет: $0
- Доход: ~$1,000/день (зависит от типа)
- Расходы: аренда, электричество, зарплаты
- Если денег нет → бизнес замораживается

## 🔐 Безопасность & Валидация

### Middleware автоматически проверяет:

✅ Существует ли пользователь в БД
✅ Находится ли пользователь в режиме выборов
✅ Приватный ли это чат
✅ Есть ли у пользователя нужная роль
✅ Является ли пользователь лидером организации

### Handler'ы дополнительно проверяют:

✅ Достаточно ли денег
✅ Правильные ли ID
✅ Есть ли доступ к операции
✅ Всё ли в порядке с БД запросом

## 🔧 Расширение функционала

### Добавить новый handler

1. Создайте файл `handlers_newfeature.py`
2. Определите Router:

```python
router = Router()

@router.callback_query(F.data == "my_button")
async def my_handler(callback: CallbackQuery, state: FSMContext):
    # Ваш код
    pass
```

3. Зарегистрируйте в `main.py`:

```python
from handlers_newfeature import router as new_router
dp.include_router(new_router)
```

### Добавить новую State

В `states.py`:

```python
class MyFeatureStates(StatesGroup):
    menu = State()
    selecting = State()
    processing = State()
```

### Добавить новую CallbackData

В `keyboards.py`:

```python
@dataclass
class MyCallback(CallbackData, prefix="myfeature"):
    action: str
    param: int
```
