## 🔧 Технические Детали

### Использованные технологии

```
┌─────────────────────────────────────────┐
│ Python 3.8+                             │
│ ├─ aiogram 3.4.1 (async framework)      │
│ ├─ aiosqlite 3.0.0 (async SQLite)       │
│ ├─ asyncio (async/await)                │
│ └─ python-dotenv (environment vars)     │
│                                         │
│ Database: SQLite (30+ tables)           │
│ FSM: aiogram.fsm.state.State            │
│ UI: InlineKeyboardMarkup + CallbackData │
└─────────────────────────────────────────┘
```

### Ключевые архитектурные решения

1. **Асинхронность везде**
    - Нет блокирующих операций
    - Все БД операции async/await
    - Polling не зависит от скорости БД

2. **Middleware для валидации**
    - EnsureUserMiddleware - создание пользователя
    - GlobalLockMiddleware - защита режима выборов
    - Расширяемая система для добавления новых

3. **FSM для диалогов**
    - Четкая последовательность состояний
    - Невозможны невалидные переходы
    - Разделение по функциям

4. **CallbackData factories**
    - Типизированные callbacks
    - Лучшая поддержка IDE
    - Безопаснее рефакторинга

5. **Модульная структура**
    - Легко добавлять новые features
    - Отсутствуют циклические зависимости
    - Возможно параллельное написание разных модулей
